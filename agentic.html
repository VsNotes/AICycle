😠 THE PROBLEM — WHY OLD AI WASN'T ENOUGH
📰 A REAL STORY FROM 2023...

Imagine you're a developer at a company. You open ChatGPT and ask:
"Research our competitors, write a report, and email it to my team."

ChatGPT says: "Sure! Here's a draft report... [but I can't actually search the web, can't access your company email, and I forgot everything from 5 minutes ago]"

😠 The old problem: LLMs were stuck in a box. They could TALK but not DO. Like hiring a genius who can only answer questions but can't pick up a phone, open a browser, or remember who you are.

✨ Before agents, developers had to:
• Manually copy-paste between ChatGPT and their tools
• Write custom code for every API integration
• Re-explain context every single conversation
• Babysit every step of a multi-step task

⚡ The Agentic AI solution: Give the LLM TOOLS (web search, email API, file access), MEMORY (remember past conversations), and a LOOP (keep trying until the job is done). Now it's not just talking — it's actually working!

🤔 WHAT IS AGENTIC AI?
🤖The Agent says:

"Don't just answer me — go figure it out!"

A normal chatbot waits for you to give it all the info. An Agentic AI takes initiative — it searches the web, reads files, runs code, calls APIs, remembers past conversations, and loops until the job is done.

Think of it like the difference between a calculator (gives answer instantly) vs a junior developer (goes off, investigates, tries things, comes back with a solution).

📦 What Each Library Does (Plain English!)

  • LangChain — builds prompts, chains LLM calls, defines tools. The "scaffolding".
  • LangGraph — runs the stateful ReAct loop. Decides when to loop, when to stop.
  • OpenAI SDK — talks to GPT-4o. LangChain wraps this automatically.
  • FAISS — stores text as vectors. Semantic "remember" for documents.
  • FastAPI — wraps the agent as a web API so anyone can call it.

from langchain_core.tools import tool from langchain_community.tools import DuckDuckGoSearchRun @tool # This decorator tells LangChain "this is an agent tool!" def get_weather(city: str) -> str: """Get current weather for a city. Use when user asks about weather.""" return f"Weather in {city}: 22C, Sunny (mock data)" @tool def calculate(expression: str) -> str: """Safely evaluate a math expression like '42*17' or '100/4'.""" try: return str(eval(expression)) except: return "Error: invalid expression" search = DuckDuckGoSearchRun() # Free web search, no API key! tools = [get_weather, calculate, search]
💡 The docstring IS the tool description!

The LLM reads your docstring to decide WHEN to call the tool. "Get current weather for a city" tells it exactly when to use it.

from langgraph.graph import StateGraph, END from typing import TypedDict class State(TypedDict): messages: list # shared state passed between agents researcher = create_react_agent(llm, [search, rag_tool]) writer = create_react_agent(llm, []) critic = create_react_agent(llm, []) g = StateGraph(State) g.add_node("researcher", lambda s: researcher.invoke(s)) g.add_node("writer", lambda s: writer.invoke(s)) g.add_node("critic", lambda s: critic.invoke(s)) g.add_edge("researcher", "writer") g.add_edge("writer", "critic") g.add_edge("critic", END) g.set_entry_point("researcher") crew = g.compile()
✅ CH 8 DONE
