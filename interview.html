🌱 ML & AI FUNDAMENTALS
💡 Answer:
AI is the broadest field — any technique that makes machines "intelligent." ML is a subset where systems learn from data instead of explicit rules. Deep Learning is a subset of ML using multi-layer neural networks.

Think of it as: AI ⊃ ML ⊃ Deep Learning

  • AI: Expert systems, rule-based bots, chess engines
  • ML: Random Forest, XGBoost, scikit-learn models
  • DL: CNNs, RNNs, Transformers (PyTorch/TensorFlow)
💡 Every LLM (GPT, Claude) is Deep Learning → ML → AI
💡 Answer:
1. Supervised Learning — labelled data (input + answer pairs). Examples: email spam, house price prediction.

2. Unsupervised Learning — no labels. Model finds hidden patterns. Examples: K-Means clustering, anomaly detection.

3. Reinforcement Learning — agent takes actions, receives rewards/penalties. Examples: AlphaGo, robotics, trading bots. 💡 In Python interviews: "sklearn for supervised/unsupervised; gymnasium for RL"
💡 Answer:
Overfitting = model memorises training data including noise, performs poorly on new data.

Prevention:
  • train_test_split + cross-validation
  • Regularisation: Ridge, Lasso, ElasticNet
  • Dropout in PyTorch: nn.Dropout(0.3)
  • Early stopping, more training data, simpler model
from sklearn.model_selection import cross_val_score model = RandomForestClassifier(100) scores = cross_val_score(model, X, y, cv=5)
💡 Answer:
A Neural Network is a series of layers (input → hidden → output), each applying a linear transform + activation function.

Backpropagation: After a forward pass, compute loss. Use chain rule to calculate gradient of loss w.r.t. every weight. Gradient descent updates weights to reduce loss.
loss = loss_fn(outputs, labels) loss.backward() # compute gradients optimizer.step() # update weights
💡 Answer:
A Transformer is a neural network architecture (2017, "Attention is All You Need") using self-attention to process all tokens in parallel.

All modern LLMs (GPT, Claude, Gemini, BERT) are Transformers. Self-attention captures relationships between any two tokens regardless of distance. 💡 Key components: Multi-head attention, positional encoding, feed-forward layers, LayerNorm
💬 LLMs & PROMPT ENGINEERING
💡 Answer:
An LLM is a Transformer trained on trillions of tokens to predict the next token. At inference:
  • Tokenises input → embeds → passes through Transformer blocks
  • Outputs probability distribution over all tokens
  • Samples next token (temperature controls creativity)
  • Repeats until end-of-sequence
💡 temperature=0 = deterministic, best for agents
💡 Answer:
Prompt engineering = crafting inputs to get the best LLM outputs.

  • Zero-shot: Direct question, no examples
  • Few-shot: Provide 2–3 examples before the question
  • Chain-of-Thought (CoT): "Think step by step" — dramatically improves reasoning
💡 Answer:
RAG = retrieve relevant docs from a vector store → inject as context → LLM answers from context. No retraining needed.

RAG vs Fine-tuning:
  • RAG: updatable, cheaper, no GPU, cites sources
  • Fine-tuning: bakes knowledge into weights, expensive, can't update without retraining
💡 RAG for factual retrieval; fine-tuning for style/behaviour changes

🌱 .NET AI FUNDAMENTALS
💡 Answer:
Semantic Kernel (SK) is Microsoft's SDK for LLM apps in C#/.NET. It provides Kernel (orchestration), Plugins ([KernelFunction] methods), Planners, and Memory.

vs LangChain: SK = Microsoft/Azure-native, C# idioms, strong typing. LangChain = Python-first, larger ecosystem. 💡 .NET/Azure team → Semantic Kernel. Python prototyping → LangChain.
💡 Answer:
Use the builder pattern — very C# idiomatic.
var kernel = Kernel.CreateBuilder() .AddOpenAIChatCompletion( modelId: "gpt-4o", apiKey: config["OpenAI:ApiKey"]!) .Build(); var result = await kernel.InvokePromptAsync( "Explain RAG in one sentence.");
💡 Answer:
A KernelPlugin is a C# class whose methods tagged [KernelFunction] become AI-callable tools.
public class WeatherPlugin { [KernelFunction] [Description("Get current weather for a city")] public string GetWeather( [Description("The city name")] string city) => $"Weather in {city}: 22°C, Sunny"; } kernel.Plugins.AddFromType<WeatherPlugin>("weather");
💡 Be specific in [Description] — the LLM reads it to decide WHEN to call this function
💡 Answer:
OpenAI direct: Simple setup, just API key. Good for prototyping.

Azure OpenAI: Your Azure subscription. VNET isolation, managed identity, compliance, data residency, SLA.
var kernel = Kernel.CreateBuilder() .AddAzureOpenAIChatCompletion( deploymentName: "gpt-4o-deployment", endpoint: "https://myco.openai.azure.com", apiKey: azureKey) .Build();
💡 Enterprise: Always recommend Azure OpenAI for compliance and Azure AD integration
