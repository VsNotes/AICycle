🌱 ML & AI FUNDAMENTALS
Q1 What is the difference between AI, Machine Learning, and Deep Learning?EasyML ▶
AI is the broadest field — any technique that makes machines "intelligent." ML is a subset where systems learn from data instead of explicit rules. Deep Learning is a subset of ML using multi-layer neural networks.

Think of it as: AI ⊃ ML ⊃ Deep Learning

  • AI: Expert systems, rule-based bots, chess engines
  • ML: Random Forest, XGBoost, scikit-learn models
  • DL: CNNs, RNNs, Transformers (PyTorch/TensorFlow)
💡 Every LLM (GPT, Claude) is Deep Learning → ML → AI
Q2 What are the 3 types of Machine Learning?EasyML ▶
1. Supervised Learning — labelled data (input + answer pairs). Model learns to predict labels for new inputs. Examples: email spam, house price prediction.

2. Unsupervised Learning — no labels. Model finds hidden patterns. Examples: customer segmentation (K-Means), anomaly detection.

3. Reinforcement Learning — agent takes actions, receives rewards/penalties. Examples: game playing (AlphaGo), robotics, trading bots. 💡 In Python interviews: "sklearn for supervised, unsupervised; gymnasium for RL"
Q3 What is overfitting and how do you prevent it in Python?EasyML ▶
Overfitting = model memorises training data including noise, performs poorly on new data. High train accuracy, low test accuracy.

Prevention in Python:
  • train_test_split + cross-validation
  • Regularisation: Ridge, Lasso, ElasticNet
  • Dropout layers in PyTorch: nn.Dropout(0.3)
  • Early stopping, more training data, simpler model
from sklearn.model_selection import cross_val_score model = RandomForestClassifier(100) scores = cross_val_score(model, X, y, cv=5) # 5-fold CV → reliable estimate of real performance
Q4 What is a Neural Network and how does backpropagation work?EasyDL ▶
A Neural Network is a series of layers (input → hidden → output), each applying a linear transform + activation function. Inspired loosely by neurons.

Backpropagation: After a forward pass, compute loss. Use chain rule to calculate gradient of loss w.r.t. every weight. Gradient descent updates weights to reduce loss.
import torch.nn as nn loss_fn = nn.CrossEntropyLoss() optimizer = torch.optim.Adam(model.parameters(), lr=0.001) loss = loss_fn(outputs, labels) loss.backward() # compute gradients optimizer.step() # update weights
Q5 What is a Transformer and why is it important?EasyDL ▶
A Transformer is a neural network architecture (2017, Google "Attention is All You Need") using self-attention to process all tokens in parallel, unlike RNNs which process sequentially.

Why important: All modern LLMs (GPT, Claude, Gemini, BERT) are Transformers. Self-attention captures relationships between any two tokens regardless of distance, enabling understanding of context. 💡 Key components: Multi-head attention, positional encoding, feed-forward layers, LayerNorm
💬 LLMs & PROMPT ENGINEERING
Q6 What is an LLM and how does it generate text?EasyLLM ▶
An LLM (Large Language Model) is a Transformer trained on trillions of text tokens to predict the next token. At inference, it:
  • Tokenises input → converts to integer IDs
  • Embeds tokens → high-dim vectors
  • Passes through stacked Transformer blocks
  • Outputs probability distribution over all tokens
  • Samples next token (temperature controls creativity)
  • Repeats until end-of-sequence
💡 temperature=0 = deterministic (same answer every time), best for agents
Q7 What is prompt engineering? Name 3 key techniques.EasyLLM ▶
Prompt engineering = crafting inputs to get the best LLM outputs.

3 Key techniques:
  • Zero-shot: Direct question, no examples
  • Few-shot: Provide 2–3 examples before the question
  • Chain-of-Thought (CoT): "Think step by step" — dramatically improves reasoning
# Few-shot example in LangChain prompt = """Classify sentiment. Positive/Negative: "Great product!" → Positive "Terrible service" → Negative "It was okay" → """
Q8 What is RAG and why is it better than fine-tuning for most use cases?EasyRAG ▶
RAG (Retrieval-Augmented Generation) = retrieve relevant docs from a vector store → inject as context → LLM answers from that context. No retraining needed.

RAG vs Fine-tuning:
  • RAG: updatable (add docs anytime), cheaper, no GPU needed, cites sources
  • Fine-tuning: bakes knowledge into weights, expensive, can't update without retraining, may hallucinate confidently
💡 Rule of thumb: RAG for factual retrieval, fine-tuning for style/behaviour changes
🐍 PYTHON & LANGCHAIN BASICS
Q9 What is LangChain and what problems does it solve?EasyLangChain ▶
LangChain is a Python framework that simplifies building LLM-powered applications. It solves:
  • Prompt management: Templates, variables, few-shot examples
  • LLM abstraction: Switch between OpenAI/Anthropic/local models easily
  • Tool integration: @tool decorator makes any function AI-callable
  • Memory: Conversation history management
  • RAG pipelines: Document loaders, text splitters, retrievers
💡 LangChain = "glue code" so you write business logic, not boilerplate HTTP calls
Q10 What is a vector embedding and why does it enable semantic search?EasyRAG ▶
A vector embedding converts text into a list of numbers (e.g. 1536 floats for OpenAI's model) where similar meanings → similar vectors (close in vector space).

Why semantic search works: "How do I get a refund?" and "refund policy 30 days" have no overlapping words but similar vectors → FAISS finds them with cosine similarity.

from langchain_openai import OpenAIEmbeddings emb = OpenAIEmbeddings() vec = emb.embed_query("What is RAG?") # vec = [0.023, -0.145, 0.892, ...] 1536 floats
⚡ LANGCHAIN & LANGGRAPH
Q1 Explain the ReAct pattern and how LangGraph implements it.MediumAgents ▶
ReAct (Reasoning + Acting) = agent loop: Think → Act (call tool) → Observe result → Think again → until done.

LangGraph implementation: StateGraph with nodes (functions) and edges (transitions). MemorySaver persists state across turns. create_react_agent handles the loop automatically.
from langgraph.prebuilt import create_react_agent from langgraph.checkpoint.memory import MemorySaver agent = create_react_agent(llm, tools, checkpointer=MemorySaver()) result = agent.invoke( {"messages": [HumanMessage("Search for AI news")]}, config={"configurable": {"thread_id": "s1"}} )
Q2 How do you implement tool calling in LangChain? What makes a good tool description?MediumLangChain ▶
Use the @tool decorator. The docstring IS the tool description — the LLM reads it to decide when to call the tool.
from langchain_core.tools import tool @tool def get_stock_price(ticker: str) -> str: """Get the current stock price for a ticker symbol. Use when the user asks about stock prices or financial data. Example: 'AAPL', 'GOOGL', 'MSFT'""" return fetch_price(ticker)
Good description: States WHEN to use it, gives examples of input format, is specific not vague. ⚠️ Vague docstring = LLM won't know when to call it!
Q3 What is the difference between FAISS and ChromaDB? When would you use each?MediumRAG ▶
FAISS: Facebook's in-memory vector library. No server, blazing fast, pure Python/C++. Best for: prototyping, single-machine deployments, read-heavy workloads.

ChromaDB: Embedded/client-server vector DB with persistence, metadata filtering, and REST API. Best for: local persistent storage, multi-user apps, metadata-rich queries.

  • Use FAISS: "Load once, query many times" — e.g. docs Q&A app
  • Use Chroma: Need to add/delete docs at runtime, need filtering
  • Use Qdrant/Pinecone: Production, multi-tenant, billions of vectors
Q4 How does chunk_size and chunk_overlap affect RAG quality?MediumRAG ▶
chunk_size controls how much text per chunk. Too small = loses context. Too large = irrelevant text dilutes the answer.

chunk_overlap ensures sentences spanning chunk boundaries aren't cut in half. 10–15% overlap is typical.

  • 512 tokens, 64 overlap → good general-purpose default
  • 256 tokens → precise factual retrieval
  • 1024 tokens → richer context but slower, more noise
💡 Always test retrieval quality with your specific documents — no universal best value!
Q5 What is LangGraph StateGraph? How is it different from a simple chain?MediumLangGraph ▶
A LangChain chain is linear: A → B → C → output. No loops, no branching, no state.

A LangGraph StateGraph is a directed graph where:
  • Nodes = functions that transform state
  • Edges = transitions (can be conditional)
  • State = TypedDict shared between all nodes
  • Supports cycles (loops) — essential for agents!
from langgraph.graph import StateGraph, END from typing import TypedDict class AgentState(TypedDict): messages: list g = StateGraph(AgentState) g.add_node("agent", agent_fn) g.add_node("tools", tools_fn) g.add_conditional_edges("agent", should_continue)
Q6 How do you serve a LangGraph agent as a REST API?MediumDeployment ▶
Use FastAPI with async endpoints. Each request gets its own thread_id for session isolation.
from fastapi import FastAPI from pydantic import BaseModel app = FastAPI() class Q(BaseModel): question: str; session: str = "default" @app.post("/ask") async def ask(q: Q): cfg = {"configurable": {"thread_id": q.session}} r = agent.invoke({"messages": [HumanMessage(q.question)]}, cfg) return {"answer": r["messages"][-1].content} # Run: uvicorn api:app --reload

Q1 Design a multi-agent system with LangGraph. How do you pass state between agents?HardMulti-Agent ▶
Each agent is a node in the StateGraph. State (TypedDict) is passed between nodes — agents read from it and write back to it.
class CrewState(TypedDict): messages: list research_results: str draft_content: str quality_score: float # Each agent has its own tools but shares state researcher = create_react_agent(llm, [search, rag_tool]) writer = create_react_agent(llm, []) critic = create_react_agent(llm, []) g = StateGraph(CrewState) g.add_node("research", lambda s: researcher.invoke(s)) g.add_node("write", lambda s: writer.invoke(s)) g.add_node("critique", lambda s: critic.invoke(s)) g.add_edge("research", "write") g.add_edge("write", "critique") g.add_edge("critique", END)
💡 Use conditional edges to loop back — e.g. if quality_score < 0.8, write again

Q3 What is the difference between semantic search and hybrid search? When would you use hybrid?HardRAG ▶
Semantic search: Cosine similarity on embeddings. Finds meaning-similar text even with different words.

Keyword search (BM25): TF-IDF style. Exact word matches, great for names, codes, IDs.

Hybrid search: Combines both with a weighted score (Reciprocal Rank Fusion). Best of both worlds.

Use hybrid when: Users search for product codes (PROD-1234) + descriptions ("comfortable running shoe"). Semantic alone misses exact codes; keyword alone misses descriptions. 💡 Qdrant and Azure AI Search both support hybrid search natively
Q4 Explain agent memory types: short-term vs long-term vs episodic.HardAgents ▶
Short-term (in-context): The current conversation messages in the LLM context window. Lost when session ends. Managed by MemorySaver + thread_id.

Long-term (vector memory): Important facts stored in a vector DB (FAISS/Qdrant). Retrieved semantically when relevant. Persists across sessions.

Episodic memory: Summaries of past interactions stored and retrieved when similar context arises. Enables the agent to "remember" previous conversations with a user.

🔥 Advanced pattern: Summarise old messages and store in vector DB when context window fills up → seamless infinite memory
Q5 How do you evaluate RAG quality? What metrics do you use?HardRAG ▶
Retrieval metrics:
  • Context Precision — are retrieved chunks actually relevant?
  • Context Recall — are all relevant chunks retrieved?
Generation metrics:
  • Faithfulness — does the answer only use retrieved context (no hallucination)?
  • Answer Relevance — does the answer address the question?
Tools: RAGAS library automates all of these using LLM-as-judge.
from ragas import evaluate from ragas.metrics import faithfulness, answer_relevancy results = evaluate(dataset, metrics=[faithfulness, answer_relevancy])
👑 SYSTEM DESIGN & ARCHITECTURE
Q1 Design a production-grade multi-tenant RAG system. What are the key architectural decisions?ProSystem Design ▶
Key decisions:

1. Tenant isolation: Separate Qdrant collections per tenant OR single collection with tenant_id metadata filter. Collections = stronger isolation; metadata = cheaper but risk of cross-tenant leaks.

2. Vector store: Qdrant (self-hosted for cost) or Pinecone (managed). At scale → Qdrant on Kubernetes with persistence.

3. Ingestion pipeline: Async queue (Celery/Redis) for document processing. Chunking, embedding, and upserting shouldn't block API responses.

4. Caching: Semantic caching (GPTCache) — if similar question asked before, return cached answer. 60-80% cost reduction at scale.

5. Observability: LangSmith or LangFuse for trace every agent call — essential for debugging and cost monitoring. 🔥 Interview tip: Always mention tenant isolation, async ingestion, and observability — shows production thinking
Q2 How do you prevent prompt injection attacks in an agentic AI system?ProSecurity ▶
Prompt injection = malicious text in user input or retrieved docs tries to override the system prompt or exfiltrate data.

Defences:
  • Input sanitisation: Strip or escape prompt-like patterns before injection
  • Privilege separation: Use separate LLM calls for untrusted content and trusted instructions
  • Tool sandboxing: Limit what tools can do — read-only by default, explicit approval for writes
  • Output validation: Validate agent output before executing (especially code execution)
  • LLM guard: Use a classifier LLM to detect injection attempts before main call
⚠️ Never run code returned by an LLM without sandboxing (Docker, subprocess with timeout)
Q3 How would you reduce LLM API costs by 70% without sacrificing quality?ProCost ▶
Strategy stack:
  • Semantic caching (GPTCache): 40-60% reduction. Similar questions → cached answer, no API call
  • Model routing: Use GPT-4o only for complex reasoning; GPT-4o-mini for simple classification/extraction (10x cheaper)
  • Prompt compression: LLMLingua — compress long context by 4-20x while preserving meaning
  • Streaming + early stop: Stop generation when answer complete, don't generate max_tokens
  • Batch API: OpenAI Batch API = 50% cheaper for non-realtime workloads
  • Chunking optimisation: Retrieve fewer, better chunks → shorter context → lower cost
💡 At 1M daily queries: semantic caching alone saves ~$15,000/month on GPT-4o pricing
Q4 You're building a coding assistant agent. Walk through your complete architecture.ProSystem Design ▶
Architecture layers:
  • Input: User's code question + repo context (tree-sitter parsed code chunks in FAISS)
  • Planner node: Classifies intent — debug, explain, generate, refactor
  • Code retriever: Semantic search on codebase for relevant files/functions
  • Code executor tool: Docker sandbox, 30s timeout, resource limits, captures stdout/stderr
  • Generator node: Writes/fixes code using context + retrieved snippets
  • Validator node: Runs linting (ruff, mypy) on generated code, loops if errors
  • Response: Code + explanation + diff from original
Stack: LangGraph StateGraph + FAISS + FastAPI + Docker executor + Redis for session state
Q5 How do you implement observability and debugging for a production LangGraph agent?ProMLOps ▶
Full observability stack:
  • LangSmith: Every LLM call, tool call, token count, latency, cost traced automatically. Set LANGCHAIN_TRACING_V2=true
  • Structured logging: Log thread_id, user_id, tool called, tokens used as JSON to CloudWatch/Datadog
  • Metrics: Track P95 latency, tokens/request, tool success rate, hallucination rate (RAGAS)
  • Alerts: PagerDuty on P95 > 5s, error rate > 2%, tool failure rate > 10%
  • A/B testing: Shadow mode — run new agent alongside old, compare outputs before full rollout

🌱 .NET AI FUNDAMENTALS
Q1 What is Semantic Kernel and how does it compare to LangChain?EasySK ▶
Semantic Kernel (SK) is Microsoft's open-source SDK for building LLM-powered applications in C#/.NET (also Python/Java). It provides:
  • Kernel = central orchestration object
  • Plugins = tool collections ([KernelFunction] methods)
  • Planners = autonomous step-by-step reasoning
  • Memory = semantic vector storage
vs LangChain:
  • SK = Microsoft/enterprise-first, Azure-native, C# idioms, strong typing
  • LangChain = Python-first, larger ecosystem, faster iteration
  • Concepts mirror each other: @tool ≈ [KernelFunction], LangGraph ≈ SK Planner
💡 If your team is .NET/Azure-heavy → Semantic Kernel. Python-first prototyping → LangChain.
Q2 How do you build and configure a Semantic Kernel in C#?EasySK ▶
Use the builder pattern — very C# idiomatic. Add LLM service, then call .Build().
using Microsoft.SemanticKernel; var kernel = Kernel.CreateBuilder() .AddOpenAIChatCompletion( modelId: "gpt-4o", apiKey: config["OpenAI:ApiKey"]!) .Build(); // Simple prompt call var result = await kernel.InvokePromptAsync( "Explain RAG in one sentence."); Console.WriteLine(result);
For Azure OpenAI: Replace AddOpenAIChatCompletion with AddAzureOpenAIChatCompletion(deploymentName, endpoint, apiKey)
Q3 What is a KernelPlugin and how do you define a KernelFunction?EasySK ▶
A KernelPlugin is a C# class whose methods tagged [KernelFunction] become AI-callable tools. The [Description] attribute = the docstring the LLM reads.
public class WeatherPlugin { [KernelFunction] [Description("Get current weather for a city")] public string GetWeather( [Description("The city name")] string city) => $"Weather in {city}: 22°C, Sunny"; } // Register with kernel kernel.Plugins.AddFromType<WeatherPlugin>("weather");
💡 Be specific in [Description] — the LLM reads it to decide WHEN to call this function
Q4 What is the difference between OpenAI and Azure OpenAI in Semantic Kernel?EasyAzure ▶
OpenAI direct: Uses OpenAI's API endpoint. Simpler setup, just API key. Good for prototyping.
var kernel = Kernel.CreateBuilder() .AddOpenAIChatCompletion("gpt-4o", apiKey) .Build();
Azure OpenAI: Deployed on your Azure subscription. Enterprise features: VNET isolation, managed identity, compliance, data residency, SLA.
var kernel = Kernel.CreateBuilder() .AddAzureOpenAIChatCompletion( deploymentName: "gpt-4o-deployment", endpoint: "https://myco.openai.azure.com", apiKey: azureKey) .Build();
💡 Enterprise answer: Always recommend Azure OpenAI for compliance, data sovereignty, and Azure AD integration

Q3 What is Semantic Kernel Process Framework and when would you use it over AgentGroupChat?HardSK ▶
