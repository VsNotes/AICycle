⚙️ HOW LLMs GENERATE TEXT
💬 LLM Bot — the magic loop:

Every word you see from ChatGPT = this loop ran once. One token at a time. A 500-word answer = ~650 loops in ~3 seconds. 🤯

1
🔢 TOKENISE

"Hello World!" splits into integer IDs: [9906, 3146, 0]
The model never reads letters — only numbers.

each number maps to...
2
🧮 EMBED

Token 9906 → [0.21, -0.87, ...] — a 4096-number vector encoding meaning.
"King" and "Queen" vectors are mathematically similar.

all tokens talk to each other...
3
👁️ ATTENTION

Every token looks at every other simultaneously across 120 stacked layers.
Figures out "IT" = trophy not bag — that's contextual understanding!

scores every possible next word...
4
🎲 SAMPLE

Final layer scores all ~100K tokens:  "Paris" 42% "London" 18%
Temperature=0: always "Paris". Temperature=1: random. 🎲

add token → go back to step 1...
5
🔁 REPEAT

The picked token joins the input and the loop runs again. Token by token until a special [END] token appears.
This is autoregressive generation.

🔄
Steps 1→5 repeat for every single token. A 500-word answer = ~650 loops. GPT-4 runs all of this in ~3 seconds. That's why GPUs matter!
🔤 KEY CONCEPTS — HOVER ANY TERM
🔢
TOKEN

Text splits into tokens. GPT-4 knows ~100,000 different tokens. Your message = list of numbers the model reads.

🧮
EMBEDDING

Each token maps to a vector of numbers encoding its meaning. Similar concepts point the same mathematical direction.

👁️
ATTENTION

The core of Transformers. Every token pays attention to ALL others at once — understands context, not just word order.

🌡️
TEMPERATURE

0 = deterministic & factual. 0.7 = creative & balanced. 1+ = unpredictable. Use 0 for coding, 0.7 for writing.

📏
CONTEXT WINDOW

Max tokens processed at once. Fill it up and the model forgets earlier parts. Bigger = better but costs more.

🌀
HALLUCINATION

LLMs predict likely-sounding text, not truth. They can generate fluent, confident, completely wrong answers.

💬 MISSION 2/10 · 🐍 PYTHON
TOKENISATION 🔢
How text splits into tokens. tiktoken library. Token counting for cost control.
😤 THE PROBLEM

LLMs don't read letters or words. They read tokens — subword chunks mapped to numbers. Get tokenisation wrong and you'll over-spend on API calls and hit context limits unexpectedly.

🔢 Token Bot:

'Hello' = 1 token. 'ChatGPT' = 2 tokens. 'pneumonoultramicroscopic...' = 13 tokens 😅

💰 Cost: OpenAI charges per token. 1M tokens ≈ $15.
📏 Limits: GPT-4 max = 128K tokens. Fill it = model forgets start.

Rule: 1 token ≈ 0.75 English words. 1000 tokens ≈ 1.5 pages. 🎯

import tiktoken # 🔢 tiktoken = OpenAI's official tokeniser enc = tiktoken.encoding_for_model("gpt-4o") text = "ChatGPT changed the world in November 2022" tokens = enc.encode(text) print(f"Text: {text}") print(f"Tokens: {tokens}") # list of ints print(f"Token count: {len(tokens)}") # 9 tokens # 💰 Estimate cost before calling API def estimate_cost(text: str, model="gpt-4o") -> float: enc = tiktoken.encoding_for_model(model) n_tokens = len(enc.encode(text)) cost_per_m = 15.0 # $15 per million input tokens return n_tokens * cost_per_m / 1_000_000 big_doc = "word " * 10000 # 10,000 words cost = estimate_cost(big_doc) print(f"Estimated cost: ${cost:.4f}") # ~$0.20 # 🎯 Decode tokens back to text for t in tokens: print(f"{t} → '{enc.decode([t])}'") # see each token!
💬 MISSION 2 COMPLETE!
  • tiktoken counts tokens accurately — use BEFORE sending to API
  • 1 token ≈ 0.75 words. 1000 tokens ≈ $0.015 for GPT-4o
  • Context window = token limit for your WHOLE conversation
  • Chunk large documents — don't send 100 pages in one API call

from openai import OpenAI import numpy as np client = OpenAI() def embed(text: str) -> list[float]: return client.embeddings.create( model="text-embedding-3-small", input=text ).data[0].embedding # 1536 numbers! def similarity(a: list, b: list) -> float: a, b = np.array(a), np.array(b) return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))) # 🧪 Test semantic similarity v1 = embed("The packaging was terrible, box arrived crushed") v2 = embed("bad packaging") v3 = embed("great product, love it") print(f"crushed box vs bad packaging: {similarity(v1,v2):.2f}") # ~0.82! print(f"crushed box vs great product: {similarity(v1,v3):.2f}") # ~0.12 # 🎯 Finds semantic match even with ZERO common words!
💬 MISSION 3 COMPLETE!
  • text-embedding-3-small: 1536-dim vectors, very cheap
  • Cosine similarity: 1.0=same meaning, 0=unrelated
  • Enables semantic search — find by MEANING not keywords
  • Core of RAG (Ch5) — embed your docs, search by question meaning

from openai import OpenAI from tenacity import retry, stop_after_attempt, wait_exponential client = OpenAI() # ⚡ STREAMING — shows tokens as they arrive def stream_chat(messages: list) -> str: full = "" with client.chat.completions.stream( model="gpt-4o", messages=messages ) as stream: for chunk in stream.text_stream: print(chunk, end="", flush=True) full += chunk return full # 💬 CONVERSATION HISTORY history = [{"role": "system", "content": "You are a helpful Python expert."}] def chat(user_msg: str) -> str: history.append({"role": "user", "content": user_msg}) reply = stream_chat(history) history.append({"role": "assistant", "content": reply}) if len(history) > 20: history[1:3] = [] return reply # 🔁 RETRY — handles rate limits automatically @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=4, max=10)) def reliable_chat(msg: str) -> str: return chat(msg)
💬 MISSION 4 COMPLETE!

  • Keep conversation history as messages[] list

from langchain_openai import ChatOpenAI, OpenAIEmbeddings from langchain_community.vectorstores import FAISS from langchain.chains import RetrievalQA # ✅ RAG — 10 lines, ready in 2 hours! docs = [ "Our refund policy: 30 days, no questions asked.", "Shipping: free over £50, 3-5 business days.", "Premium members get 2-day delivery and priority support." ] emb = OpenAIEmbeddings(model="text-embedding-3-small") vs = FAISS.from_texts(docs, emb) chain = RetrievalQA.from_chain_type( llm=ChatOpenAI(model="gpt-4o"), retriever=vs.as_retriever(search_kwargs={"k": 2}) ) print(chain.invoke("What's the refund policy?")["result"]) # "30 days, no questions asked." — accurate, cited! 🎯

from openai import OpenAI import json client = OpenAI() tools = [{ "type": "function", "function": { "name": "get_weather", "description": "Get current weather for a city.", "parameters": { "type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"] } } }] def get_weather(city: str) -> str: return f"{city}: 22°C, Sunny" messages = [{"role": "user", "content": "What's the weather in London?"}] response = client.chat.completions.create( model="gpt-4o", tools=tools, messages=messages) tool_call = response.choices[0].message.tool_calls[0] args = json.loads(tool_call.function.arguments) result = get_weather(**args) print(f"Result: {result}") # London: 22°C, Sunny ✅

from fastapi import FastAPI from fastapi.responses import StreamingResponse from openai import OpenAI app = FastAPI(title="💬 LLM Chat API") client = OpenAI() sessions: dict[str, list] = {} @app.post("/chat/{session_id}") async def chat(session_id: str, message: str): if session_id not in sessions: sessions[session_id] = [ {"role": "system", "content": "You are a helpful assistant."} ] sessions[session_id].append( {"role": "user", "content": message}) def generate(): full = "" with client.chat.completions.stream( model="gpt-4o", messages=sessions[session_id] ) as stream: for chunk in stream.text_stream: full += chunk yield f"data: {chunk}\n\n" sessions[session_id].append( {"role":"assistant","content":full}) return StreamingResponse(generate(), media_type="text/event-stream") # 🚀 uvicorn llm_api:app → test at /docs
